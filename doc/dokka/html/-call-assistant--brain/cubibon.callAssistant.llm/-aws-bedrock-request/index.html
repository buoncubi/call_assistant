AwsBedrockRequest

data class AwsBedrockRequest(val prompts: List<SystemContentBlock>, val messages: List<Message>, val maxTokens: Int = System.getenv(ENV_VAR_AWS_BEDROCK_MAX_TOKENS).toInt(), val temperature: Float = System.getenv(ENV_VAR_AWS_BEDROCK_TEMPERATURE).toFloat(), val topP: Float = System.getenv(ENV_VAR_AWS_BEDROCK_TOP_P).toFloat(), val modelName: String = System.getenv(ENV_VAR_AWS_BEDROCK_MODEL_NAME)) : LlmRequest<List<SystemContentBlock>, List<Message>>

The implementation of LlmResponse for AWS Bedrock, which should be given to AwsBedrock.computeAsync.

Author

Luca Buoncompagni, © 2025, v1.0.

Parameters

prompts

The list of prompts.

messages

The list of messages. Each item in the list encodes the message role, and the last item should have the USER role.

maxTokens

The maximum number of tokens that the LLM model can produce. By default, it is set through the environmental variable with a name equal to the ENV_VAR_AWS_BEDROCK_MAX_TOKENS constants.

temperature

The temperature to configure the LLM model. By default, it is set through the environmental variable with a name equal to the ENV_VAR_AWS_BEDROCK_TEMPERATURE constants.

topP

The top-probability to configure the LLM model. By default, it is set through the environmental variable with a name equal to the ENV_VAR_AWS_BEDROCK_TOP_P constants.

modelName

The LLM model to be used. By default, it is set through the environmental variable with a name equal to the ENV_VAR_AWS_BEDROCK_MODEL_NAME constants.
